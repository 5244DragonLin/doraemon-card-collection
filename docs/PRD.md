### 1.1 项目背景

已有一个单 HTML 文件的基础版本，实现了卡包浏览、稀有度分组、拥有标记、水平排列预览、搜索、导入导出。但存在以下核心不足：

1. **圆环预览是伪 3D**：现有 carousel 仅用 `translateX + scale` 做水平直线排列 + 缩放，不是真正的 3D 圆环/Coverflow 效果，"左右渐退消失"不明显
2. **色调偏冷**：主色 `--primary: #4A90D9`（冷蓝），Header 用 `linear-gradient(135deg, #4A90D9, #357ABD)`，不符合"温暖、积极向上"的品牌要求
3. **动画偏少**：仅有铃铛摆动、进度条流光、卡牌入场等基础动画，整体偏静态
4. **数据生成脚本缺陷**：
   - 正则 `^([A-Za-z?]+)-(.+)$` 无法处理中文前缀特殊命名（如 `特殊SSP-夏威夷风情精雕卡`、`EX内页-哆啦A梦勋章书卡`），导致 **80 张卡牌**被错误归类为 `?` 未知级别
   - 仅扫描 `.png` 文件，**遗漏 453 张 `.jpg` 文件**（实际卡牌总数应为 ~1708 张，而非当前的 1255 张）

## 2. 产品目标

**一句话**：打造一个温暖、灵动、展示性强的卡动文创图鉴，以真正的 3D Coverflow 圆环预览为核心亮点，让收藏爱好者享受沉浸式的卡牌浏览与收藏管理体验。

**核心目标**：

1. **视觉升级**：从冷蓝色调全面切换到温暖积极的哆啦A梦主题色调（铃铛金、竹蜻蜓蓝绿、温暖橙等），整体明亮活泼，符合"温馨搞笑"作品调性
2. **3D 圆环预览**：实现真正的 3D Coverflow 效果（基于 CSS 3D `perspective + rotateY` 变换），卡牌沿弧形排列，左右渐退消失，支持鼠标/触摸横向拖拽流畅浏览
3. **数据完整性**：修复数据生成脚本，正确识别所有特殊命名卡牌的级别，同时扫描 `.png` 和 `.jpg` 文件，确保零遗漏
4. **动效丰富化**：在页面各处增加恰到好处的动画（卡牌悬浮、切换过渡、拥有标记弹跳、背景动效等），让网站"活"起来但不喧宾夺主
5. **收藏体验**：强化收藏展示特性，提供清晰的收藏进度可视化、便捷的标记操作、可靠的数据持久化与备份恢复

### 5.3 3D 圆环预览弹窗（核心升级点）

**视觉描述**：

```
│  ◀ 拖拽浏览同级别卡牌 ▶    3 / 12        │
└──────────────────────────────────────────┘
```

**3D 效果技术要点**（供架构师参考）：

- 容器设置 `perspective: 1000px` 建立 3D 空间
- 每张卡牌根据距中心偏移量计算变换：
  - `rotateY(±角度)`：越远角度越大（如每偏移 1 张旋转 35°），形成弧形排列
  - `translateZ`：中心卡牌向用户方向推出，远处卡牌后退
  - `scale`：远处缩小
  - `opacity`：远处降低至 0.2~0.3 后完全渐退消失
  - `filter: blur()`：远处轻微模糊增强景深
  - `z-index`：中心最高
- 拖拽时实时更新所有卡牌位置（跟随手指），松手后用 `transition` 平滑吸附到最近卡牌
- 中心卡牌带金色高光边框 + 阴影，突出当前选中

### 5.4 动画清单

| 位置 | 动画 | 触发 | 效果 |
|---|---|---|---|
| Header 铃铛 | 摆动 | 持续 | `rotate(±15deg)` 2s 循环 |
| 卡牌入场 | 上滑淡入 | 滚动进入视口 | `translateY(20px) → 0` + `opacity 0→1`，0.5s |
| 卡牌悬浮 | 上浮放大 | hover | `translateY(-4px) scale(1.05)` + 阴影增强 |
| 拥有标记 | 星标弹跳 | 标记时 | `scale(0→1.3→1)` 0.4s + 金色光圈 |
| 进度条 | 流光 | 持续 | 渐变色背景位移循环 |
| 弹窗 | 缩放淡入 | 打开/关闭 | `scale(0.9→1)` + `opacity`，0.3s |
| 圆环切换 | 3D 旋转过渡 | 切换卡牌 | 所有卡牌 `transition: transform 0.4s` 平滑过渡到新位置 |
| 背景 | 微动元素 | 持续 | 缓慢飘浮的半透明图案（可选 P1） |
